# Weldo Android app (weldo_android)

Jetpack Compose client for **Weldo**, an AI-based todo/task app. Talks to the same Spring Boot
backend as the iOS (`../doit_ui`) and web (`../doit_web`) clients — see `../doit_back/JSON.md`
for the wire contract and `../doit_back/DOMAIN.md` for the domain model. When the domain changes
on the backend, mirror it here too.

## Stack

Generated from an Android Studio (preview) "New Project", then the Weldo client was layered on:

- **Kotlin 2.2.10**, **Jetpack Compose** (Material 3), **AGP 9.2.1 / Gradle 9.4.1**, compile/target SDK **37**, min SDK **32**. Java 11 source level.
- AGP 9 **bundles the Kotlin plugin** — the app applies only `com.android.application`, `kotlin.plugin.compose`, and (added) `kotlin.plugin.serialization`. Do **not** add `org.jetbrains.kotlin.android` (double-apply).
- **Retrofit + OkHttp + kotlinx.serialization** networking; **Coil** images; **DataStore** session; **Navigation Compose**.
- **Manual DI, no Hilt/KSP** — everything hangs off `WeldoApp.graph` (`AppGraph`); the build has no annotation processors.
- MVVM: no-arg `ViewModel`s that read `WeldoApp.graph.repository`; screens get them via `viewModel()` and collect with `collectAsState()`.

## Build & run

Open in Android Studio and Run, or CLI (needs a JDK 17–21 — AGP 9 rejects newer; the repo builds with Temurin 21):

```sh
cd weldo_android
JAVA_HOME=<jdk-21> ./gradlew :app:assembleDebug   # → app/build/outputs/apk/debug/app-debug.apk
JAVA_HOME=<jdk-21> ./gradlew :app:installDebug    # onto a running emulator/device
```

`local.properties` (SDK path) is generated by Studio and git-ignored.

## Talking to the backend

- **Base URL** is configurable at runtime (Login → Advanced, or Settings → Server), persisted in DataStore. Default `http://10.0.2.2:5555` = the Android **emulator's** alias for the host's `localhost:5555` (`./gradlew localAnthropicBootRun`). Physical device → point at the ngrok https tunnel. Cleartext http is manifest-allowed for the emulator.
- **Auth** (`WeldoSession` holds the bearer, an HMAC JWT ~30-day TTL):
  - Email/password → `POST /auth/password/{login,register}`.
  - **Dev token** → Login → Advanced → type a name → sends `Authorization: Bearer test-<name>`, accepted by the backend's `local` profile. Fastest local path.
  - **Sign in with Apple** → web OAuth (Custom Tab) delivering Apple's `id_token` to `weldo://apple-auth` → `POST /auth/apple`. **Scaffolding**: fill `APPLE_WEB_CLIENT_ID`/`APPLE_REDIRECT_URI` (buildConfigFields in `app/build.gradle.kts`) plus a hosted redirect forwarding the token to the custom scheme. Backend code-exchange is a stub, so the token must arrive as an `id_token`.
- **Wire quirks mirrored** (don't fix one side only — see `data/model/Models.kt`): `/task` singular vs plural; `waiting_for` type vs `waiting-for` path; completion serializes as `completed` (also decodes `isCompleted`); Routine flag is `active`; every timestamp is a raw `String` (backend emits microsecond Java `Instant` + `LocalDate` days) parsed by `data/remote/WeldoTime.kt`; `PUT /{id}` is full-replace, completion via `PUT /<type>/{id}/completed`.

## Architecture

- `data/model/` — wire models (all 9 types + `AllItems`, `CaptureProposal`, batch, auth, profile, stats) + `ItemType` (endpoint/color/completable) + `AnyItem` (type-erased row model).
- `data/remote/` — `WeldoApi` (every endpoint), `ApiProvider` (rebuilds Retrofit on base-URL change; per-request bearer/Apple headers), `WeldoTime`.
- `data/local/WeldoSession.kt` — DataStore token / Apple-user / base-URL / stats-period.
- `data/WeldoRepository.kt` + `ItemDraft.kt` — the single data gateway; `ItemDraft` is a unified editable shape spanning all nine types so **one** form and **one** detail screen drive every type.
- `ui/` — `WeldoNavHost` (routes), `MainShell` (the signed-in shell: bottom `NavigationBar` Home/Items/Stats + a compact `CaptureBar` accessory + Settings gear — the web left rail translated to a bottom bar), `theme/` (`WeldoTheme`), `ItemVisuals` (`ItemType.icon`/`.color`), `common/` (shared `ItemRow`, dialogs, states), then features: `login/` (email/pw + dev token + Apple), `home/` — **`AgendaScreen`** (Home tab: open task/project/reminder items bucketed by due date — Overdue→Today→Tomorrow→This week→This month→Later, empty buckets dropped, ported from web `core/due.ts`; greeting shows overdue/today) + **`ItemsScreen`** (All-items tab: filter chips + search + mixed list) — both content-only, driven by one shared `HomeViewModel` (`agenda()` builds the buckets); `capture/` (CaptureBar: `compact` = the `＋ Capture` menu accessory / full = new+AI buttons; text/voice/photo → `/capture` review + batch sheets; `ItemFormSheet`; `VoiceRecorder`), `detail/`, `project/`, `settings/`, `stats/` (`StatsPane` = the tab body; `StatsScreen` wraps it with a back bar for the Settings link). `auth/` holds the Apple redirect plumbing.

## Status

- **Builds green**: `:app:assembleDebug` succeeds (JDK 21) with zero warnings → `app-debug.apk`. Not yet run on a device/emulator against a live backend — runtime behavior unverified.
- Voice uses the system `SpeechRecognizer` (needs Google's recognition service); no on-device fallback like iOS. No offline cache. Note markdown renders as plain text. Date pickers are day-only (no time-of-day for `remindAt`/`followUpAt`, which default to UTC midnight).
- The Studio template left `ui/theme/Color.kt`/`Type.kt`/`Theme.kt` (`MyApplicationWeldoTheme`) unused; `WeldoTheme` is the live one.
